date: 2025-01-18

---


First impressions set expectations. This post outlines how to onboard users to agent-powered features.

## Progressive disclosure

Don’t dump all capabilities at once. Start with one clear task (e.g. "Summarize this") and reveal more (e.g. "Suggest edits", "Compare versions") as the user gains confidence.

## Set expectations

Explain what the agent does and what it doesn’t: "It will suggest edits; you decide what to accept." Avoid hype; under-promise and over-deliver so trust builds over time.

## Safe first run

Design the first run so the user can’t break anything critical. Use a draft or sandbox mode, and make the path to "publish" or "apply" explicit and reversible where possible.
