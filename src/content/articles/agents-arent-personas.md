## Hook

Eileen's Slack message linking to Parallel.ai: "y'all, get ready to design for the newest persona, Agents."

## Thesis

Calling agents a "persona" is a category error that will lead designers to apply the wrong methods, create the wrong artifacts, and build the wrong products. Agents have specifications, not psychology.

## Key Points

- What personas actually are — archetypal humans with goals, frustrations, mental models.
- Why persona methodology doesn't apply to agents — no psychology, no empathy maps, no journey maps.
- What Parallel.ai actually is — infrastructure for agents to consume, not UX patterns.
- The real design problems — API design for agent interactions, dev tooling for humans building agents, control interfaces for humans working with agents.

## Payoff

Introduction to why we need new frameworks. The old tools don't work on new terrain.
